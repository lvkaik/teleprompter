# CI/CD 使用指南（GitHub Actions）

> 配套：`.github/workflows/` + `scripts/generate-keystore.sh`

本项目已经配置好 GitHub Actions 自动打包。**不需要本机装 Android Studio**，只要把代码推到 GitHub，就能自动产出 APK。

---

## 1. 快速开始（5 步）

### Step 1 — 推到 GitHub

```bash
cd /Users/lvkai/Documents/ticiqi
git init
git add .
git commit -m "feat: 初始版本"
git branch -M main
git remote add origin https://github.com/你的用户名/teleprompter.git
git push -u origin main
```

### Step 2 — 触发 workflow

推上去后自动触发，也可以手动：

```
GitHub Repo → Actions → Build APK → Run workflow → Run
```

### Step 3 — 等待约 5-8 分钟

CI 会自动：
1. 拉代码
2. 装 JDK 17
3. 装 Android SDK（首次约 5min，后续用缓存 30s）
4. 编译 debug APK
5. 上传为 Artifact

### Step 4 — 下载 APK

```
GitHub Repo → Actions → 选刚跑完的 run → 底部 Artifacts
→ 下载 teleprompter-debug-apk.zip
→ 解压得到 app-debug.apk
```

### Step 5 — 装到手机

```bash
adb install app-debug.apk
# 或者直接手机打开文件管理器点 APK 安装
```

**注意：debug APK 必须用 debug 签名才装得上**（CI 自动签好了）。

## 2. 两个 workflow 区别

| 文件 | 触发条件 | 产出 | 用途 |
|------|---------|------|------|
| `build.yml` | push / PR / 手动 | debug APK (默认) | 日常调试、自测 |
| `release.yml` | 仅手动 | release APK + AAB + GitHub Release | 正式发布 |

## 3. 怎么产出 release APK（带签名的正式包）

release APK 用了你**自己生成的 keystore**，需要配置 GitHub Secrets。

### 3.1 生成 keystore

```bash
cd /Users/lvkai/Documents/ticiqi
./scripts/generate-keystore.sh
```

按提示输入两次密码（keystore 密码和 key 密码），脚本会：
- 生成 `release.jks`
- 生成 `keystore.properties`
- 把 base64 编码写到 `keystore-base64.txt`
- 自动更新 `.gitignore`

### 3.2 ⚠️ 立刻备份 release.jks

```
这个文件丢了就再也签不出同样签名的 APK，
意味着 Google Play 上架后所有用户都得重新装新版本。
强烈建议：
  - 1Password / Bitwarden
  - 加密 U 盘
  - 至少 2 个独立备份点
```

### 3.3 配置 GitHub Secrets

进入 `GitHub Repo → Settings → Secrets and variables → Actions → New repository secret`：

| Secret 名 | 值 |
|----------|---|
| `RELEASE_KEYSTORE_BASE64` | `keystore-base64.txt` 整段内容 |
| `RELEASE_KEYSTORE_PASSWORD` | keystore 密码 |
| `RELEASE_KEY_ALIAS` | `teleprompter` |
| `RELEASE_KEY_PASSWORD` | key 密码 |

### 3.4 触发 release

```
Actions → Release → Run workflow
  version_name: 1.0.0
  version_code: 1
  prerelease: false
→ Run workflow
```

完成后会自动：
- 构建 release APK
- 构建 release AAB（Google Play 用）
- 在 GitHub Releases 创建 `v1.0.0`

## 4. 本地打包（可选）

如果你装了 Android Studio，也可以本地打包：

```bash
# Debug
./gradlew assembleDebug
# → app/build/outputs/apk/debug/app-debug.apk

# Release（需要 keystore.properties）
./gradlew assembleRelease
# → app/build/outputs/apk/release/app-release.apk
```

## 5. CI 大概耗时

| 阶段 | 首次 | 后续（带缓存） |
|------|------|--------------|
| 拉代码 | 5s | 5s |
| 装 JDK | 10s | 0s |
| 装 SDK / 命令行工具 | 60s | 0s |
| 下载 Gradle 依赖 | 5-10min | 30s |
| 编译 | 3-5min | 3-5min |
| 打包 | 30s | 30s |
| **合计** | **8-15min** | **5-7min** |

> 缓存策略：基于 commit hash 自动缓存，下次只编译差异部分。

## 6. 故障排查

### Q: CI 跑挂了 "SDK location not found"

A: workflow 已自动装 SDK，不需要你手动配。如果还报错，删除本地 `local.properties` 后重推。

### Q: release 构建报 "keystore was tampered with, or password was incorrect"

A: 检查 Secrets 里的 4 个值是否与生成时一致。

### Q: 下载的 APK 装不上 / 解析失败

A: 大概率是下载中断，校验文件大小（一般 debug 约 8-12 MB）。

### Q: 想跑 instrumentation test 但没设备

A: 当前 workflow 没启用，需要自建 firebase test lab 或加 device farm。

### Q: 不想每次 push 都构建

A: 编辑 `.github/workflows/build.yml`，删掉 `push:` 触发器，只留 `pull_request` 和 `workflow_dispatch`。

## 7. 进阶

### 7.1 自动发版到内测分发平台（如 fir.im / 蒲公英）

在 `release.yml` 末尾加：

```yaml
- name: Upload to pgyer
  env:
    PGY_API_KEY: ${{ secrets.PGY_API_KEY }}
  run: |
    curl -F "file=@app/build/outputs/apk/release/app-release.apk" \
         -F "_api_key=$PGY_API_KEY" \
         https://www.pgyer.com/apiv2/app/upload
```

### 7.2 自动提交到 Google Play

加 `r0adkll/upload-google-play@v1` action，需配置 Google Play Console API service account JSON。

### 7.3 自动发版到 Telegram 群

加 `appleboy/telegram-action@master` action。

### 7.4 矩阵构建（多 minSdk 验证）

```yaml
strategy:
  matrix:
    min-sdk: [26, 29, 33]
```

## 8. 安全提示

| 项 | 建议 |
|----|------|
| release.jks | 至少 2 个独立备份点，**永不提交到 git** |
| keystore.properties | 加入 `.gitignore`，**永不提交** |
| GitHub Secrets | 定期轮换（建议每 90 天换一次密码） |
| 发布 tag | 用语义版本（v1.0.0、v1.1.0），不要用"最新" |
| 二步验证 | 给 GitHub 账号开 TOTP |
