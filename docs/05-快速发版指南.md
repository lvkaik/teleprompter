# 🚀 快速发版：5 步拿到 APK

> 目标：从 0 开始，5-10 分钟拿到能在 realme GT7 Pro 上跑的悬浮提词器 APK

---

## 第 1 步 · 在 GitHub 上建一个空仓库（1 分钟）

浏览器打开：**https://github.com/new**

| 字段 | 填什么 |
|------|--------|
| Repository name | `teleprompter` |
| Description | `Android 悬浮提词器，支持 AI 语音驱动` |
| Public/Private | 任选（Private 也行） |
| **Add README** | ❌ **不要勾** |
| Add .gitignore | ❌ 不要勾 |
| Add license | ❌ 不要勾 |

点 **Create repository**。

记下你的 GitHub 用户名，比如 `lvkai`。

---

## 第 2 步 · 推送代码（1 分钟）

打开 Terminal，粘贴：

```bash
cd /Users/lvkai/Documents/ticiqi
./scripts/push-to-github.sh 你的GitHub用户名 teleprompter
```

例：`./scripts/push-to-github.sh lvkai teleprompter`

脚本会：
- 初始化 git
- 写入所有文件
- 提交
- 推送到 GitHub

> 如果提示要登录 GitHub，按提示操作（推荐用 SSH key，比 token 简单）。

---

## 第 3 步 · 等待 CI 自动跑（约 8 分钟）

打开：**https://github.com/你的用户名/teleprompter/actions**

你应该能看到 **"Build APK"** 正在运行（黄色旋转）。

| 阶段 | 大约耗时 |
|------|---------|
| 装 JDK | 10 秒 |
| 装 Android SDK | 60 秒 |
| 拉 Gradle 依赖 | 5 分钟（首次） |
| 编译 APK | 3 分钟 |
| 打包上传 | 30 秒 |
| **合计** | **~8 分钟** |

等顶部那个黄色圆圈变成绿色 ✅。

---

## 第 4 步 · 下载 APK（30 秒）

点击进入这次 run → 滚动到底部 **Artifacts** 区域：

```
teleprompter-debug-apk
```

点击下载（一个 zip 文件）→ 解压得到 `app-debug.apk`（约 10 MB）。

---

## 第 5 步 · 装到 GT7 Pro（1 分钟）

三种方法任选：

### 方法 A：USB 拷过去（推荐）

```bash
# 1. 用数据线连 GT7 Pro（开启 USB 调试，传输文件模式）
# 2. 复制
cp ~/Downloads/app-debug.apk /Volumes/手机存储/Download/

# 3. 手机文件管理器打开 Download 文件夹 → 点 app-debug.apk → 安装
```

### 方法 B：AirDrop / 微信传给自己

把 `app-debug.apk` 拖到微信"文件传输助手"或者隔空投送到手机 → 手机打开 → 安装。

> ⚠️ 微信可能会改后缀成 `.apk.1`，改回来即可。

### 方法 C：scrcpy / adb install

如果有 `adb`：

```bash
adb install ~/Downloads/app-debug.apk
```

---

## 装完后第一次启动要做

1. 弹窗"允许通知" → **允许**
2. 进入 App，菜单 → "后台保活设置"
3. 按引导把 4 项都做了（自启动 / 电池优化 / 应用冻结 / 锁定任务卡片）
4. 进入 App → 设置 → 填入 MiniMax API Key
5. 回到主页 → 创建一条文稿
6. 点文稿右侧的 **▶ 按钮** → 悬浮窗启动
7. 长按悬浮窗顶部可拖动，拖四个角可缩放
8. 点悬浮窗内的播放按钮开始滚动

---

## 🎉 完事

现在你有一个 **能跑、能调、能装到真机的悬浮提词器**，全流程没碰过 Android Studio。

---

## 常见问题

### Q：推 GitHub 时报 "Repository not found"

A：第 1 步没建仓库，或者名字拼错了。再检查一下 GitHub 上的 repo 名。

### Q：CI 失败 "SDK location not found"

A：极少数情况 workflow 缓存异常，去 Actions → 失败 run → 右上 "Re-run jobs" 重跑。

### Q：APK 装不上 "解析失败"

A：大概率下载中断，重新下载 zip 解压重试。

### Q：装上了但启动就闪退

A：多半是 `targetSdk=36` 太高被你系统挡了，去 App 详情看崩溃日志。或者用 logcat：
```bash
adb logcat | grep -i "teleprompter\|AndroidRuntime"
```

### Q：想改代码后重新打包

A：随便改一个 `.kt` 文件，然后：
```bash
git add . && git commit -m "fix: xxx" && git push
```
等 3-5 分钟（增量编译），去 Actions 下载新的 APK。
