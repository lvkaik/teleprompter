# 悬浮提词器 — 需求文档（PRD）

> 项目代号：**Floating Teleprompter**
> 版本：v1.0
> 编写日期：2026-07-21
> 目标机型：realme GT7 Pro（realme UI 7.0 / Android 16）

---

## 1. 项目概述

### 1.1 一句话定义

一款 Android 端的悬浮提词器应用，让用户在直播、录视频、演讲、采访等场景下无需手持稿件，通过可悬浮、可移动、可调整大小的透明提词框实时显示文稿；并支持 AI 语音识别驱动自动滚动，彻底解放双手。

### 1.2 项目目标

- 提供比市面提词器 App 更稳定的悬浮窗体验（针对 realme UI 7.0 深度适配）
- 通过 AI 语音识别实现"说哪滚哪"的零操作体验
- 极简 UI，新用户 3 步内开启首次提词

### 1.3 目标用户

| 用户角色 | 使用场景 | 核心痛点 |
|---------|---------|---------|
| 短视频/直播创作者 | 边走边播、镜头前口播 | 看稿时眼神飘忽、忘词 |
| 讲师 / 培训师 | 课程录制 | 翻页打乱节奏 |
| 商务 / 销售 | 路演、发布会 | 紧张忘词 |
| 普通用户 | 视频录制、面试准备 | 需要自然流畅的口播 |

## 2. 功能需求（Functional Requirements）

### 2.1 FR1 — 文稿编辑与管理

| 编号 | 功能 | 优先级 |
|-----|------|-------|
| FR1.1 | 新建/编辑/删除提词文稿 | P0 |
| FR1.2 | 文稿列表（标题、字数、最后修改时间） | P0 |
| FR1.3 | 富文本编辑（粗体、字号、颜色高亮关键词） | P1 |
| FR1.4 | 文稿导入（TXT、Markdown、剪贴板粘贴） | P1 |
| FR1.5 | 文稿导出与分享 | P2 |
| FR1.6 | 字数统计与预计朗读时长（按语速估算） | P1 |
| FR1.7 | 文稿分类/标签/搜索 | P2 |

### 2.2 FR2 — 悬浮窗提词（核心 P0）

| 编号 | 功能 | 详情 |
|-----|------|------|
| FR2.1 | 悬浮窗启动 | 通过悬浮窗按钮一键开启，可叠加在任何 App 之上 |
| FR2.2 | **自由拖动** | 长按标题栏可拖动到屏幕任意位置 |
| FR2.3 | **自由调整大小** | 四角/四边把手拉伸，支持最小 200×150dp、最大屏宽×0.95 倍高度 |
| FR2.4 | 透明度调节 | 0.2 – 1.0 无级调节 |
| FR2.5 | 字体大小调节 | 12 – 48 sp |
| FR2.6 | 颜色与背景 | 文字色、背景色、行间距 |
| FR2.7 | 镜像翻转 | 适配专业提词器硬件 |
| FR2.8 | 单行/多行模式 | 单行超高亮，多行段落 |
| FR2.9 | 阅读位置记忆 | 关闭后下次打开从上次位置继续 |
| FR2.10 | 应急隐藏 | 一键隐藏 / 一键恢复 |

### 2.3 FR3 — 滚动控制

| 编号 | 功能 | 详情 |
|-----|------|------|
| FR3.1 | 手动滚动 | 上下滑动、点击顶部/底部按钮 |
| FR3.2 | **匀速自动滚动** | 1 – 200 px/s 滑块调速 |
| FR3.3 | **AI 智能滚动（核心亮点）** | 见 FR4 |
| FR3.4 | 暂停/继续 | 悬浮按钮一键控制 |
| FR3.5 | 进度条显示 | 显示当前位置 / 总字数 |
| FR3.6 | 跳转 | 点击进度条快速跳转 |

### 2.4 FR4 — AI 智能提词（基于 MiniMax）

| 编号 | 功能 | 详情 |
|-----|------|------|
| FR4.1 | 麦克风收音 | 16kHz PCM 实时采集 |
| FR4.2 | **语音识别 (ASR)** | 实时流式识别用户口播内容（MiniMax API） |
| FR4.3 | **文稿匹配** | 将识别文本与提词文稿做模糊匹配（编辑距离 / 滑动窗口） |
| FR4.4 | **自动滚动到当前朗读位置** | 匹配成功后自动推进 |
| FR4.5 | **断句纠错** | 错读、漏读、重复读容错 |
| FR4.6 | **敏感词过滤与用户词表** | 自定义专业术语读音 |
| FR4.7 | AI API Key 配置 | 用户在设置中填入 MiniMax API Key |
| FR4.8 | 离线兜底 | 无网络时降级为手动 + 匀速滚动 |

### 2.5 FR5 — 系统与设置

| 编号 | 功能 |
|-----|------|
| FR5.1 | 开机自启（可选） |
| FR5.2 | 通知栏快捷开关 |

## 3. 非功能需求

| 类别 | 指标 |
|------|------|
| 性能 | 悬浮窗渲染 CPU < 3%；滚动跑满 120fps（GT7 Pro）；冷启动 < 1.5s |
| 兼容性 | Android 9.0 – 16；realme GT7 Pro 优先验证 |
| 稳定性 | 连续运行 4 小时无崩溃；悬浮窗保活 > 95% |
| 隐私 | 麦克风数据本地处理或加密上传；不上传原始音频到云端（仅 ASR 流） |
| 电量 | 后台运行时功耗 < 8%/h |
| 安全 | API Key 使用 EncryptedSharedPreferences 存储 |
| 易用性 | 用户从安装到首次悬浮提词 ≤ 3 步 |

## 4. 用户故事（User Stories）

> **US1** 作为一名短视频创作者，我想在打开相机录制的同时开启一个半透明悬浮提词框，让我的眼神一直看着镜头却能看到稿件。

> **US2** 作为创作者，我希望提词框可以拖到镜头正上方，并缩到合适大小，使画面更自然。

> **US3** 作为讲师，我希望匀速滚动的速度可以微调，以匹配我的语速。

> **US4** 作为创作者，我希望应用可以"听到"我说的话，并自动滚动到我正在读的位置，这样我完全不用手动控制。

> **US5** 作为用户，我希望在设置中配置我自己的 MiniMax API Key，并管理用量。

> **US6** 作为 realme GT7 Pro 用户，我希望应用在 realme UI 7.0 的后台管理下也能稳定存活。

---

## 5. 验收标准（关键）

1. 在 GT7 Pro 上悬浮窗可在抖音、相机、WhatsApp 等任意 App 之上显示。
2. 悬浮窗拖动无卡顿，调整大小边界不溢出屏幕。
3. AI 模式下，对普通话识别准确率 > 92%，匹配推进误差 < 1 行。
4. 关闭悬浮窗后再次打开，能恢复到上次的位置、字号、透明度。
5. 网络断开时自动切换为本地匀速滚动模式，无功能中断。
6. realme UI 7.0 上连续挂机 1 小时，应用未被系统杀进程。
